\section{Implementation}

\subsection{Code: Pipeline Explanation}
The 3D scene reconstruction pipeline was implemented using Python, NumPy, and Open3D. Below is a detailed breakdown of the methodologies applied at each stage:

\begin{enumerate}
    \item \textbf{Data Loading \& Depth Quantization Fix:} 
    The original \texttt{load.py} script compressed continuous depth data into an 8-bit integer format (0-255). This caused severe quantization error, resulting in jagged, stepped walls. I modified the data collection script to save depth as a 16-bit unsigned integer (\texttt{uint16}) multiplied by a scale of 1000.0. This preserved sub-millimeter geometric accuracy and adhered to the specification's \texttt{DEPTH\_SCALE = 1000.0}.

    \item \textbf{Task 1: Geometric Unprojection:}
    To convert the RGB-D images into a 3D point cloud, I utilized the Pinhole Camera Model. Instead of using computationally expensive \textit{for-loops}, I optimized the function using \texttt{np.meshgrid} to map the $(u, v)$ coordinates and utilized vectorized NumPy operations to calculate the $(X, Y, Z)$ spatial coordinates instantly. The Y-axis was intentionally inverted (\texttt{-y}) to align the computer vision image space with Habitat's 3D world space.

    \item \textbf{Pre-processing:}
    Raw point clouds contain over 260,000 points per frame. I applied a Voxel Downsample with a size of \texttt{0.05m} to drastically reduce computation time while preserving critical geometry (corners and edges). I then estimated surface normals (required for Point-to-Plane ICP) and computed Fast Point Feature Histograms (FPFH) to create unique local geometric signatures for global registration.

    \item \textbf{Registration Pipeline:}
    \begin{itemize}
        \item \textbf{Global Registration (RANSAC):} Used to find the initial coarse alignment. To combat "Feature Aliasing" in indoor environments (where RANSAC confuses identical flat walls and flips the room), I enforced a \texttt{CorrespondenceCheckerBasedOnNormal}. This strict constraint forced the algorithm to only match points facing the exact same physical direction.
        \item \textbf{Local Registration (ICP):} Used Open3D's Point-to-Plane ICP to take RANSAC's initial transformation and strictly lock the flat surfaces together using a tight threshold (\texttt{0.05m}) to prevent odometry drift.
    \end{itemize}

    \item \textbf{Post-processing (Dynamic Ceiling Removal):}
    Instead of hardcoding a ceiling height (which breaks across different floors), I implemented a data-driven approach. The script extracts the Y-coordinates of the upper half of the room and calculates a histogram. The largest density spike in the histogram dynamically identifies the exact height of the ceiling plane, allowing the script to cleanly slice off the roof regardless of the robot's starting elevation.
\end{enumerate}
